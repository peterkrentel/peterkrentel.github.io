Weatherford Centro MPD

Lead Platform Architect & DevOps Engineer · Weatherford (via Umbrage)
SHEET: A-01
PLATFORM: AWS EKS
DATE: Nov–Dec 2025

A production EKS platform for Weatherford's Centro MPD application, built on a reusable Terraform module library — the foundation later reused across other platform work.

The Challenge

No reusable path to production

Weatherford needed Centro MPD running on cloud-native infrastructure across dev, staging, and production, with enterprise-grade authentication and a CI/CD pipeline the engineering team could actually use — without requiring VDI access. Nothing about the environment provisioning process was repeatable, so every new environment meant rebuilding from scratch.

Solution Architecture

A module library, not a one-off cluster

Built a modular Terraform library — VPC, EKS, and supporting infrastructure — designed for replication rather than a single bespoke environment. Keycloak handles multi-tenant authentication; Sealed Secrets keeps encrypted secrets safely inside Git for a GitOps workflow. A custom Azure DevOps agent module let the pipeline run without VDI dependency. FluxCD reconciles every environment from Git via Kustomize/Helm overlays.

Key Accomplishments

Outcomes

  • Environment provisioning cut from 2–3 weeks to 4–8 hours
  • Terraform module library reused directly on later platform work
  • Multi-tenant Keycloak authentication with Sealed Secrets for GitOps-safe secrets management
  • Custom Azure DevOps agent module removed VDI dependency for the engineering team

Lessons Learned

What held up

Modular Terraform paid off almost immediately once a second environment needed provisioning. Deciding on Keycloak early avoided an authentication rework later. Documentation written alongside the build, not after it, is what let the pattern get picked up on other projects without me in the room.
