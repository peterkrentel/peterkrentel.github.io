American Airlines

Cloud Platform Architect & Technical Advisor · American Airlines (via Umbrage)
SHEET: A-03
PLATFORM: AWS · Azure
DATE: Apr–Jun 2025

A multi-cloud platform recommendation for American Airlines, backed by working proofs of concept on both AWS and Azure rather than a slide-deck comparison.

The Challenge

A decision that needed evidence, not opinions

American Airlines needed to choose a cloud platform strategy for application modernization, and the executive review process required a defensible, line-item cost comparison — not vendor pricing pages restated as a recommendation.

Solution Architecture

Two working pilots, not two diagrams

Rather than compare AWS and Azure on paper, I built a full-stack pilot — FastAPI backend and React frontend — deployed to Azure AKS via GitHub Actions, alongside a Terraform Cloud–provisioned AWS EKS environment. Paired both with a detailed TCO breakdown across compute, storage, networking, and operations so the final recommendation carried real numbers back to the executive review.

Multi-cloud evaluation (simplified)

FastAPI + React→ GitHub Actions → Azure AKS
Terraform Cloud→ AWS EKS
both→ cost + feasibility comparison→ executive recommendation

Lessons Learned

What held up

A working pilot moved the conversation faster than any deck could — stakeholders could click through something real instead of taking a diagram on faith. Cost comparisons only land when broken down to line items people already recognize from their own budgets, not aggregated into a single number.
